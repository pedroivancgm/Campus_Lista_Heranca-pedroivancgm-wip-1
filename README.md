# Campus_Lista_Heranca
Implementação de herança de POO no script de campusLista

A implementação ocorreu através da adição da subclasse "Monitor" na superclasse "Aluno"

Para testar a implementação siga o passo-a-passo:\
1 - Selecione o campus\
2 - Crie um aluno\
3 - Crie um monitor\
4 - Liste todos os monitores\
5 - Remova o monitor

As ações de Criar, listar e remover dão um controle básico sobre os monitores de cada campus.

===========================================================================\
A implementação do dia 03/12/2025 consistiu em utilizar de uma IA generativa para refinar o código, garantindo que 
o código esteja seguindo os conceitos de verificação dinâmica.

A maneira que eu fiz isso foi o uso de "isInstance" ao selecionar a opção de listar alunos no menu.
O isInstace verifica se o aluno é um monitor, e se caso for, mostra na tela uma informação a mais sobre o aluno "Monitoria"

Para testar isso, siga esse passo a passo:\
1 - selecione o campus\
2 - cadastre um aluno\
3 - liste os alunos (antes dele se tornar monitor)\
4 - cadastre um monitor (usando o mesmo aluno ou outro)\
5 - liste novamente os aluno e veja o que mudou.

OBS: Quando há a pergunta sobre os horários disponíveis, o código espera uma string, ou seja, digite uma frase como "das 13:00 às 15:00 nas quartas-feiras"

O código foi aprimorado com a ajuda do ChatGPT
